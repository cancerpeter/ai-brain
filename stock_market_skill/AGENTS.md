# AGENTS.md

## 项目目标

本项目用于封装股票行情类已有插件能力，提供一个轻量、可复用、可持续扩展的 skill / tool 工程骨架。

## 长期工作规则

1. 修改代码前，先阅读目录结构与相关模块职责。
2. 优先保持轻量实现，默认使用标准库，不要无端引入重依赖。
3. 优先使用规则路由，不要过早引入复杂模型或多阶段编排。
4. 保持输入、路由、数据模型、输出格式分层清晰。
5. 对外输出必须简洁、结构化、可维护，不直接暴露底层原始 JSON。
6. 新增工具时，必须同步补齐：
   - 路由规则
   - 统一 client 封装
   - 格式化逻辑
   - README / examples / skill 文档
   - 至少一条测试
7. 发生歧义时，不要瞎猜证券标的，应返回候选提示或澄清信息。
8. 发生空数据或插件异常时，优先返回友好说明，不要抛给上层生硬错误。
9. 保持函数职责单一，避免把路由、调用、格式化写在同一个函数里。
10. 如果未来接入真实插件，不要伪造“真实数据来源已验证”的描述；未接入前必须明确标注为 placeholder 或 mock。

## 代码风格要求

- Python 版本以当前环境为准
- 优先使用 `dataclasses`
- 注释适量，只解释不直观的逻辑
- 函数名、字段名统一且可读
- 避免魔法字符串散落，常量集中定义

## 修改优先级

1. 先保证路由准确和输出统一
2. 再提升参数抽取能力
3. 最后再考虑更复杂的智能化增强
